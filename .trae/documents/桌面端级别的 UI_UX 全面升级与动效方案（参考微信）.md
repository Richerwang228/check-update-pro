## 目标与原则
- 去掉常驻书签侧栏，突出“一键检查全部”的主流程。
- 参考微信桌面端的简洁层次：顶部工具栏 + 主内容区域 + 底部状态栏。
- 引入顺滑动效与骨架屏，降低等待枯燥感。

## 新布局结构
1. 顶部工具栏（AppBar）
- 左：应用标题与图标。
- 中：搜索框（过滤更新项，支持关键词与时间范围）。
- 右：图标按钮（刷新、筛选、日志、统计、设置、添加/导入书签）。
- 书签管理以对话框形式打开（新增/导入弹窗），不占据主界面。

2. 主内容区域
- 使用 `QStackedWidget` 管理视图：
  - 更新列表视图（默认）：`QListView + QAbstractListModel + QStyledItemDelegate`
  - 统计视图：卡片式统计信息（缓存、请求、封禁、活跃度等）
  - 日志视图：最近日志（可过滤/导出）
  - 设置视图：开关与范围配置（并发、缓存 TTL、动效开关）
- 列表支持：骨架屏、懒加载图片、项淡入/滑入动效、点击打开视频。

3. 底部状态栏
- 显示并发、速率限制命中、失败/封禁域名数、网络状态提示。
- 长任务期间显示顶部细进度条与半透明覆盖层，同时状态栏显示预计剩余时间与速度。

## 动效与等待体验
1. 列表项动效：
- 新增项淡入 (`opacity` 200ms) + 轻微滑入 (`geometry` y 从 +6 到 0)，缓动 `OutCubic`。
- 悬停微缩放 (1.00→1.02, 150ms) 与阴影层级提升。

2. 骨架屏与 shimmer：
- 数据待加载时显示灰色骨架条目；用 `QTimer` 驱动渐变带平移实现 shimmer。

3. 进度覆盖层（代替长期弹窗）：
- 顶部细进度条（2–4px）与半透明遮罩淡入淡出，正文可浏览；
- 显示当前任务名、百分比、预计剩余时间、提示语（循环切换，降低枯燥）。

4. 按钮微交互与反馈：
- 涟漪点击动画（叠加层圆形 alpha 与半径动画，200ms）；
- 成功/错误以右上角 toast 展示，自动消失；
- 长等待插入轮播 tips（网络/反爬建议）。

## 视觉统一与主题
- 设计令牌：spacing 8/12/16，圆角 8/12，阴影 0/1/2，字体 `Microsoft YaHei UI`/`Inter`。
- QSS 清理：减少大面积渐变，采用干净纯色 + 规范 hover/active；滚动条变细，色彩统一。
- 图标：引入 SVG/字体图标（Material/Bootstrap），与按钮状态统一。

## 技术实现点
- 替换 `QScrollArea` 为 `QListView`（更新列表）；模型与委托实现项渲染与动效。
- 顶部 AppBar、底部状态栏与 `QStackedWidget` 主视图综合管理。
- 骨架屏组件与动画工具函数抽象（`utils/animations.py`、`ui/widgets/skeleton_item.py`）。
- 设置项新增动效开关与主题切换（减少低配设备负担）。
- 保留现有后端并发与缓存优化，无需改动数据逻辑。

## 风险与控制
- 动效过多影响性能：限制动画时长与次数；提供关闭动效的设置。
- 列表大量项：采用虚拟化，只渲染可见项；图片懒加载并发受控。

## 验收标准
- 主流程“一键检查全部”更显眼，操作路径更短；
- 长任务中进度与骨架屏顺滑，覆盖层与状态栏信息清晰；
- 视觉整洁，层次明确，交互反馈轻快。

确认后我将先落地 AppBar + 进度覆盖层与骨架屏，再替换列表为 `QListView + Delegate`，最后统一 QSS 与动效工具函数，确保每步均可回退。
